`@udecode/react-utils` 包含用于 React 的实用函数。

### PortalBody

使用 portal 在 `document.body` 中渲染 React 组件。

### Text

用于渲染 span 的通用组件。

### Box

用于渲染 div 的通用组件。

### createPrimitiveComponent

创建一个原始组件工厂，它使用钩子来管理状态、props，并将引用转发到子组件。

<APIParameters>
  <APIItem name="element" type="React.ElementType">
    基础组件或原生 HTML 元素。
  </APIItem>
</APIParameters>
<APIReturns>
  返回一个带有状态和 props 管理钩子的原始组件的函数。
</APIReturns>

### createPrimitiveElement

从元素类型创建组件。

<APIParameters>
  <APIItem name="element" type="React.ElementType">
    原生 HTML 元素。
  </APIItem>
</APIParameters>
<APIReturns>
  渲染指定元素的函数组件。
</APIReturns>

### createSlotComponent

创建一个 Slot 组件，当你想要一个组件表现得像它的子组件时很有用。

<APIParameters>
  <APIItem name="element" type="React.ElementType">
    基础组件或原生 HTML 元素。
  </APIItem>
</APIParameters>
<APIReturns>

返回一个 Slot 组件的函数。

</APIReturns>

### withProviders

将组件包装到多个 providers 中的函数。

<APIParameters>
<APIItem name="providers" type="any[]">
用于包装组件的 providers。

如果你想让 provider 接收任何 props，你可以简单地传递一个数组。

</APIItem>
<APIItem name="WrappedComponent" type="FunctionComponent<T>">
要被包装的组件。
</APIItem>
<APIItem name="props" type="T">
要传递给被包装组件的 props。
</APIItem>
</APIParameters>
<APIReturns>
一个被指定 providers 包装的新组件。
</APIReturns>
