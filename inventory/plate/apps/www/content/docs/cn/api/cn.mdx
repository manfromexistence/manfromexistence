`@udecode/cn` 包含用于 React 和 Tailwind 的实用函数。

### cn

在不产生冲突的情况下有条件地添加 Tailwind CSS 类。

<APIParameters>
  <APIItem name="...inputs" type="CxOptions">
    使用 `clsx` 和 `tailwind-merge` 设置的类值。
  </APIItem>
</APIParameters>
<APIReturns>

`className` 字符串。

</APIReturns>

### withCn

为组件设置默认的 `className`。

<APIParameters>
  <APIItem name="Component" type="React.ComponentType">
    将要添加 props 的组件。
  </APIItem>
  <APIItem name="...inputs" type="CxOptions">
    使用 `cn` 设置的默认 `className`。
  </APIItem>
</APIParameters>
<APIReturns>

包含默认 `className` 的新组件。

</APIReturns>

### withProps

为组件设置默认 props。

<APIParameters>
  <APIItem name="Component" type="React.ComponentType | ElementType">
    将要添加 props 的组件。
  </APIItem>
  <APIItem name="props" type="Partial<T>">
    要添加到组件的 props。
  </APIItem>
</APIParameters>
<APIReturns>

包含默认 props 的新组件。

</APIReturns>

### withVariants

使用 `class-variance-authority` 的变体为组件设置默认 `className`。

<APIParameters>
  <APIItem name="Component" type="React.ComponentType | ElementType">
    将要添加 props 的组件。
  </APIItem>
  <APIItem name="variants" type="V extends ReturnType<typeof cva>">
    作为默认 `className` 的变体。
  </APIItem>
  <APIItem name="onlyVariantsProps" type="(keyof VariantProps<V>)[]">
    从 `Component` 中排除的 props。设置仅用于变体的 props。
  </APIItem>
</APIParameters>
<APIReturns>

包含默认 `className` 的新组件。

</APIReturns>
